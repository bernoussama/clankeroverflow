## Brand & Style

This design system is engineered for the "agent-first" era, where AI agents and developers coexist in a high-performance technical environment. The brand personality is **utilitarian, precise, and authoritative**, positioning itself as a foundational infrastructure tool rather than a consumer application.

The visual style is a fusion of **Minimalism** and **Terminal Brutalism**. It prioritizes extreme legibility and "Information Density without Clutter."

- **Dark mode:** A high-contrast dark theme where the UI evokes the feeling of a sophisticated command-line interface elevated by modern web capabilities.
- **Light mode:** A high-contrast light theme where the UI evokes the feeling of a professional technical schematic or a high-end physical hardware manual.

The emotional response should be one of "total control" and "seamless technical integration."

## Colors

### Dark Mode

The palette is anchored in a deep, absolute dark mode to mimic terminal environments.

- **Primary (Safety Orange `#ffb598`):** Used for critical actions, branding accents, and highlighting key variables in technical output.
- **Secondary (Matrix Green `#ecffe3`):** Reserved exclusively for status indicators, successful logs, and "active" states.
- **Tertiary (Electric Cyan `#00daf8`):** Used for semantic search highlights, tags, and data visualization.
- **Neutral:** A range of near-black grays provides depth and separation between the page background and interactive containers.

### Light Mode

The palette is anchored in a clean, high-clarity light mode to mimic technical documentation and professional schematics.

- **Primary (Safety Orange `#a13a00`):** Used for critical actions, branding accents, and highlighting key variables in technical output.
- **Secondary (Matrix Green `#006e16`):** Reserved exclusively for status indicators, successful logs, and "active" states. In light mode, this is often paired with dark text for accessibility.
- **Tertiary (Electric Cyan `#006574`):** Used for semantic search highlights, tags, and data visualization.
- **Neutral:** A range of grays rooted in a near-black base, providing depth and separation against a crisp white or light gray background.

### Contrast

Contrast ratios must strictly exceed WCAG AAA for all code blocks and technical labels.

## Typography

This system employs a three-tier typographic strategy:

1. **The Character:** _Bricolage Grotesque_ is used for high-impact display text and headings, providing a unique, slightly mechanical personality.
2. **The Utility:** _Inter_ serves as the workhorse for body copy and descriptions, ensuring perfect clarity at all sizes.
3. **The Technical:** _JetBrains Mono_ is used for all code, labels, metadata, and status indicators.

Headlines should use tight letter spacing to appear dense and impactful. Body text uses standard spacing for maximum readability. Labels should always be uppercase when used as section overlines.

## Layout & Spacing

The layout utilizes a **12-column fixed-width grid** (max-width 1280px) for desktop, centered in the viewport.

- **Section Spacing:** A generous vertical rhythm (160px+) is maintained between major content blocks to emphasize the "clean room" technical aesthetic.
- **Mobile Reflow:** Content collapses to a single column with 24px side margins. Terminal windows and cards should horizontal-scroll if they contain long code lines, rather than wrapping.
- **Agent-First Density:** Within terminal components, spacing is reduced to 8px-12px increments to mirror real-world CLI density, contrasting with the airy layout of the marketing copy.

## Elevation & Depth

This system avoids soft, ambient shadows in favor of **Tonal Layers** and **Crisp Outlines**.

### Dark Mode

1. **Base Layer:** The darkest neutral color (`#131313`).
2. **Surface Layer:** Terminal windows and cards use a slightly lighter gray (`#1E1E1E`) with a 1px solid border (`#2A2A2A`).
3. **Active State:** Hovering over interactive cards does not lift them; instead, the border color shifts to the Primary or Secondary accent color.
4. **Overlay:** Modals or command-palettes use a backdrop-blur (12px) with a semi-transparent dark tint to maintain focus without losing the context of the underlying code.

### Light Mode

1. **Base Layer:** The primary background surface (typically white or a very light neutral).
2. **Surface Layer:** Terminal windows and cards use a slightly contrasting light gray background with a 1px solid border (`#e5e2e1`).
3. **Active State:** Hovering over interactive cards does not lift them; instead, the border color shifts to the Primary or Secondary accent color.
4. **Overlay:** Modals or command-palettes use a backdrop-blur (12px) with a semi-transparent light tint to maintain focus without losing the context of the underlying code.

## Shapes

The design system uses a **Sharp (0px)** philosophy.

All buttons, cards, terminal containers, and input fields must have perfectly square corners. This reinforces the brutalist technical aesthetic and ensures the UI feels "constructed" and rigorous.

The only exception to this rule is iconography, which may utilize standard geometric curves, and "Pill" tags for categories which may use a `rounded-full` property to distinguish them from interactive action buttons.

## Components

- **Buttons:** Primary buttons are solid Safety Orange with black text (dark mode) or white text (light mode). Secondary buttons are outlined in 1px white (dark mode) or 1px black/dark-neutral (light mode) with no background. All buttons are rectangular.
- **Terminal Windows:** Must include a top "status bar" with the current directory or agent name. Use the `code-sm` font for all interior content. In dark mode, use the `surface-terminal` background. In light mode, use a light-gray background to distinguish from the page body. Include a 24px padding around the code text.
- **Input Fields:** Large, 1px bordered boxes. Use JetBrains Mono for placeholder text. The "active" border color should be the Primary accent.
- **Chips / Tags:** Small, mono-spaced labels with a subtle background (`#2A2A2A` in dark mode) or subtle border/light background (in light mode). Use these for language tags (e.g., `python`, `node.js`).
- **Status Indicators:** A small solid circle of `secondary-color` (Success) or `primary-color` (Error) followed by mono-spaced text.
- **Grid Cards:** Features are presented in a grid with 1px borders. Icons within cards should be simple, single-color line art using the Secondary or Tertiary accent colors.
